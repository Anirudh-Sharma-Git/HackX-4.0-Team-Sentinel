Build a clean React (Vite) + Tailwind dashboard UI for a Distributed File System project called:

COSMEON FS-Lite

IMPORTANT:
- Use ONLY real backend endpoints.
- Do NOT generate mock data.
- Do NOT simulate CPU, disk, or performance metrics.
- Do NOT create historical charts unless backend provides time-series data.
- No cluster selector.
- No fake health score.
- No placeholder analytics.
- No lorem ipsum.
- Keep UI minimal, professional, and realistic.

Tech Stack:
- React (Vite)
- Tailwind CSS
- Axios
- Lucide React icons

Backend Base URL:
http://localhost:8000

Available Endpoints:

GET    /health
GET    /nodes
GET    /files
GET    /files/{file_id}
POST   /upload
GET    /download/{file_id}
GET    /verify/{file_id}
POST   /repair
POST   /nodes/{node_id}/fail
POST   /nodes/{node_id}/recover

--------------------------------------------
UI REQUIREMENTS
--------------------------------------------

1) Cluster Status Panel
- Fetch from GET /health
- Display:
  - system_status (HEALTHY / DEGRADED / CRITICAL)
  - total_chunks
  - healthy_chunks
  - under_replicated_chunks
  - corrupted_chunks
- Show colored badge:
  - Green = HEALTHY
  - Yellow = DEGRADED
  - Red = CRITICAL

--------------------------------------------

2) Node Grid
- Fetch from GET /nodes
- Display each node:
  - node_id
  - status (ONLINE / OFFLINE)
  - chunk_count
- Show Fail button if ONLINE
- Show Recover button if OFFLINE
- Buttons must call real API endpoints.

--------------------------------------------

3) File Manager
- Fetch from GET /files
- Display table:
  - file_name
  - file_size
  - total_chunks
  - file_id
- Buttons:
  - Download → GET /download/{file_id}
  - Verify → GET /verify/{file_id}
- Show verification result clearly (PASS / FAIL)

--------------------------------------------

4) Upload Panel
- Drag & drop or file input
- POST to /upload
- After success:
  - Refresh files
  - Refresh health
  - Show upload summary

--------------------------------------------

5) Repair Button
- If system_status != HEALTHY:
  Show “Repair Cluster” button
- Calls POST /repair
- Refresh health + nodes after repair

--------------------------------------------

DESIGN REQUIREMENTS
--------------------------------------------

- Dark theme
- Clean layout
- No excessive gradients
- No fake animations
- Minimal cards with subtle shadow
- Clear spacing
- Professional system dashboard feel
